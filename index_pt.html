Introdução O que você vai encontrar neste pacote e o para que serve

Obrigado por comprar um template do Magento. Essa documentação consiste em diversas partes e demonstra todo o processos de como iniciar e administrar uma loja Magento do zero. Fizemos o máximo para manter este manual o mais claro e fácil de seguir possível.

O que é o Magento Commerce?

Magento é uma plataforma de eCommerce rica em recursos e desenvolvida com tecnologia open-source. Ela provém controle sobre o conteúdo, a aparência e a funcionalidade de sua loja de eCommerce. A interface intuitiva do Magento oferece ferramentas poderosas de marketing, busca, otimização de busca e gerenciamento de catá-logo para entregar ao comerciante o poder de criar sites de acordo com suas necessidades únicas. Desenhado para ser completamente escalonável e segurado pela rede de suporte Varien, o Magento oferece à empresas a solução de eCommerce definitiva. Mais informações

O que é um tema do Magento?

Um tema de Magento é um o design da sua loja online contruindo em cima do Magento. Isso quer dizer que você pode facilmente alterar a aparência da sua loja instalando um novo tema em poucos passos simples. Para garantir toda essa simplicidade, um tema de Magento contém todos os arquivos necessários para você extender e alterar o quanto quise.

Estrutura de arquivos

O pacote do tema que você baixou consiste de diversos diretórios. Vejamos o que cada pasta contém:

  • documentation - contém os arquivos de documentação.
    • documentation.html - a documentação principal que você está lendo agora.
  • screenshots - onde tem imagens de como seu tema se parecerá. Não é indicado para colocar no servidor de produção.
  • sources - contém os arquivos fonte do tema.
    • psd - com os arquivos fonte do Adobe Photoshop usados na criação do tema (.psd).
    • demo - contém páginas e blocos adicionais que são providos junto com o tema.
    • sample_data - contém os arquivos de dados de amostra do tema.
      • media - onde você encontra arquivos e diretórios que deveriam ser enviados para o diretorio raiz da instalação do Magento.
      • dump.sql.gz - o arquivo de recuperação do banco de dados. Contém o conteúdo de amostra do tema.
  • theme### - contém os arquivos do tema do magento e diretórios que deveriam ser enviados para o diretório raiz de sua instalação do Magento.
  • fonts_info.txt - contém os links de onde as fontes personalizadas do template podem ser baixadas.
  • info.txt - contém instruções de como extrair o código fonte.

 

Instalação completa

Você pode usar esse método se você não tiver uma loja Magento e quiser instalá-la do zero. No fim, você terá o programa do Magento, o tema, os dados de amostra e todas as extensões necessárias instalados.

 

Instalar o tema

Você pode usar esse método se você tem uma loja do Magento e quer mudar sua aparência instalando um novo tema. No fim, você terá o tema do Magento e todas as extensões necessárias instaladas.

 

 

Preparando-se

Antes de prosseguir com a instalação de sua loja Magento, verifique se você está preparado. Siga os passos abaixo::

Software de Edição

Para trabalhar com templates do Magento, recomendamos baixar todas as aplicações necessárias. Você pode verificar a lista de software necessário na visualização do template.
Os requerimentos variam entre templates, mas para a maioria deles você vai precisar de:

  1. Primeiro, você vai precisar do WinZip 9+ (para Windows) ou do Stuffit Expander 10+ (para Mac) para extrair o conteúdo protegido por senha do arquivo sources_#########.zip.
  2. Você também deve precisar do Adobe Photoshop. Ele é usado para editar arquivos .PSD, e é necessário se você quiser editar gráficos e imagens do template.
  3. Para editar arquivos de código fonte, você vai precisar de um editor de códigos como Adobe Dreamweaver, Notepad++, Sublime Text, etc.
  4. Para subir arquivos para o servidor de hospedagem você vai precisar de um gerenciador de FTP como Total Commander, FileZilla, CuteFTP, etc.

Hospedagem

O Magento Commerce é um programa desenvolvido com PHP/MySQL, você precisa preparar um ambiente de hospedagem para rodá-lo.

Se você tem uma hospedagem que você administra, certifique-se de que ela está de acordo com os requisitos mínimos do Magento.

Do contrário, você pode rodar o Magento localmente no seu computador usando um servidor local. Para criar um servidor local, use os aplicativos próprios para isso, como WAMP, AppServ, MAMP etc. Qualquer um desses pode ser facilmente instalado como qualquer outro programa e usá-lo para rodar o Magento.

SPlease check the following tutorials on how to configure the local development environment:

  • How to install AppServ web development environment
  • How to install WAMP web development environment
  • How to install XAMP web development environment

Instalação completa

O método de instalação completa pode ser usado se você precisar montar sua loja Magento usando o conteúdo de amostra. Assumimos que você já se preparou.

O processo de instalação consiste de 3 fases:

  1. Criar um banco de dados.
  2. Subir todos os arquivos necessários para o servidor.
  3. Instalar o Magento junto com o template que você comprou.

Vamos começar!

1. Criar um banco de dados

Primeiro, você precisa preparar um novo banco de dados para sua loja Magento. Você pode criar um novo banco de dados usando uma ferramenta de gerencimaneto de bancos de dados (normalmente, o PhpMyAdmin).

Os seguintes dados serão recebidos ao criar seu banco de dados: o nome do banco, nome de usuário, senha e o nome do seu servidor. Você precisa guardar esses dados para continuar a instalação do Magento.

Com a ferramenta phpMyAdmin, você pode criar um novo banco de dados em 3 passos simples:

  1. Clique no botão do banco de dados na navegação superior do menu para acessar a lista de bancos de dados
  2. Insira o nome do banco em "Create new database".
  3. Clique no botão Criar à direita do campo que você preencheu.

Seu banco de dados agora pode ser visto na listagem. Clique no nome dele para conseguir acesso ao banco:

 

  • 1. Abra a aba "Database", você verá uma lista de bancos de dados.

  • 2. Insira o nome do seu banco de dados
    3. Clique em "Create"

  • 4. Seu banco apareceu na lista. Para entrar no banco, clique no nome na lista.

 

Você pode assistir um tutorial em vídeo detalhado em How to create a database

Se você tiver algum problema, entre em contato com seu provedor de hospedagem.

2. Importar dados de amostra

O pacote do template do Magento contém um arquivo de dados de amostra. Esse arquivo está localizado em /sources/sample_data, dentro do diretório do seu tema. Com ele você pode incluir os dados de amostra para que seu site fique igual à demonstração que você viu em nosso site.

Não faça a importação do arquivo dump.sql.gz se você já tiver dados no seu site. Se fizer isso, todos os produtos já cadastrados serão apagado. Se não quiser instalar os dados de amostra, use um banco de dados vazio ao instalar o Magento. Pule o passo de instalação doss dados de amostra.

Para instalar o arquivo dump, siga esses passos:

  1. Entre no PhpMyAdmin e escolha o banco que você usará para o Magento (ele foi criado por você nos passos anteriores).
  2. Navegue até a aba "Import" e clique no botão "Browse".
  3. Encontre a pasta do "/sources/sample_data" e escolha o arquivo dump.sql.gz.
  4. Clique em "GO". O processo de importação pode demorar uns minutos.

 

  • 1. Escolha o banco que você vai usar para o Magento.
    2. Escolha "IMPORT".
    3. Clique em "Browse".

  • 4. Navegue até "/sources/sample_data" e escolha dump.sql.gz file.

  • 5. Clique em "GO".

 

3. Baixar o programa do site oficial

Para instalar o Magento, você precisa baixar o arquivo no site oficial www.magentocommerce.com/download.

Escolhe o tipo de arquivo no menu drop-down e clique em Download. Você vai precisar se registrar no site para baixar o Magento.

Certifique-se de baixar uma versão compatível com o seu template. Você pode encontrar mais informações sobre as versões do Magento na mesma página, na aba Release Archives.

4. Extrair arquivos

Quanto terminar de baixar, você vai precisar extrair os arquivos do Magento para uma pasta.

Isso pode ser feito com um gerenciador de arquivos como o WinZip(PC), StuffitExpander(MAC), etc.

Se tiver problemas, você pode ler os seguintes tutoriais: WinZip(PC) and StuffitExpander(MAC).

Perceba que: se seu painel de hospedagem tiver uma opção para 'Extrair' arquivos, você pode usá-la para extrair diretamente após o envio. Isso vai acelerar o processo de envio também.

5. Enviando os arquivos do Magento

Se você extraiu os arquivos no seu computador, envie-os ao seu servidor.

Isso pode ser feito tanto pelo painel de gerenciamento de arquivos, quanto por uma aplicação para gerenciamento de FTP como: FileZilla, TotalCommander, CureFTP, etc.

Se tiver problemas, assista os seguintes vídeos: How to upload files to a server using an FTP manager and How to upload files to a server using cPanel (WebHost Manager, WHM).

6. Subir arquivos do tema

O próximo passo é instalar o template/tema. O tema deve ser instalado, quer esteja usando os dados de amostra ou não.

  1. Em primeiro lugar, abra o pacote do template e vá para a pasta theme###.
  2. Envie o conteúdo da pasta theme### para o diretório de instalação do Magento (envie "app", "skin" and other folders). These folders contain the theme files. They will not affect your existing files.
  3. Se você á importou o arquivo dump.sql.gz você também precisa enviar as imagens dos produtos. Envie o diretório "media" dentro de sources/sample_data para a pasta raíz do Magento.
  4. Se você não instalar o arquivo dump.sql.gz file, não precisa enviar a pasta "media".

7. Instalar o Magento

Quando terminar de subir os arquivos, você pode começar a Instalar o Magento. Abra seu navegador e digite seu 'domínio/caminho para o diretório do Magento'. Você deve ver a tela de Boas vindas. Siga as instruções abaixo para instalar o Magento.

Perceba: o arquivo dump.sql.gz deve ser importado para o banco ANTES de instalar o Magento.
Step 1: Assistente de Instalação

Leia o acordo de linceção e marque a caixa "I agree to the above terms and conditions" se concordar com os termos.

Step 2: Assistente de instalação (Localização)

Escolha o Local geográfico, Fuso Horário e a moeda padrão. O Local geográfico define a língua usada pelo Magento na área administrativa. Se quiser mudar a língua do site, você vai precisar baixar o e instalar o Magento frontend locales.Todas essas opções podem ser facilmente alteradas no painel administrativo mais tarde.

Step 3: Configuração

Aqui você precisa incluir os detalhes do banco de dados da loja e as configurações iniciais. Database Connection – insira o nome de hospedagem, usuário, senha e nome do banco. Entre em contato com seu provedor de hospedagem se não tiver os detalhes necessários.

Certifique-se de ter um banco de dados criado para a instalação do Magento. Tenha certeza de que ele está limpo também, sem tabelas dentro. Web access options e Session Storage Options – deixe essas opções sem mudanças se não tiver certeza. As configurações padrão costumam funcionar sem problemas.

Se tudo estiver certo, clique no botão 'Continue'.

Step 4: Criar conta de administrador

Aqui você precisa inserir detalhes pessoais para o dono da loja e administrador.

Quando terminar, clique em 'Continue'.

Step 5: Tá tudo certo

Esse é o passo final da instalação. Copie a chave de encriptação e mantenha-a em um local seguro. Você pode usar os botões nos cantos inferiores direito e esquerdo para ver sua loja e o painel administrativo.

Instalar o template do Magento

Certifique-se de que sua instalação do Magento corresponde aos requisitos mínimos do template. Se estiver usando uma versão desatualizada do Magento, vai precisar atualizá-lo. Antes disso, não se esqueça de copiar seus arquivos e seu banco de dados. Instruções para atualizar o Magento podem ser encontradas no site Magento Wiki.

Se sua instalação do Magento estiver dentro dos requisitos do template, você pode começar a instalação.

  1. Abra o arquivo do template e vá para a pasta theme###.
  2. Envie a pasta theme### e seu conteúdo para o diretório de instalação do Magento (envie "app", "skin" e outras pastas). Essas pastas contém os arquivos do tema, não vão alterar os arquivos existentes.

1. Configurar template

Vamos repassar as opções de configuração do template baseadas no exemplo. Está configurado da mesma maneira que nossa demonstração, sem afetar os dados dos produtos.

Ativar novo template
  1. Abra o painel administrativo do Magento e navegue para System > Design no menu superior.
  2. Clique em "Add Design Change".
  3. No campo "Custom Design", escolha o nome do tema (por exemplo, theme###).
  4. Clique em "Save"para confirmar as alterações.

 

Reindexar dados
  1. Abra o painel do Magento e no menu superior escolha System > Index Management.
  2. Marque a caixa "Select All" no topo esquerdo.
  3. No campo "Actions" escolha "Reindex Data".
  4. Clique em "Submit".
  5. Aguarde, esse processo pode demorar.

 

Desabilitar cache

Para fazer as alterações aparecerem no seu site imediatamente, você precisa desabilitar o cache. Isso previne limpezas adicionais de cache após cada alteração que você fizer. Você pode habilitá-lo novamente quando terminar de editar sua loja.

  1. Abra o painel do Magento e vá para System > Cache Management.
  2. No topo à esquerda, clique em "Select All".
  3. No lado direito, no campo "Actions" escolha "Disable".
  4. Clique no botão "Submit".

 

2. Configurar logotipo.

Siga os passos abaixo para definir sua imagem de logotipo, inclusa no pacote do tema. (você pode vê-lo na página de demonstração).

  1. Abra o painel do Magento e navegue para System > Configuration.
  2. Clique na aba "Design".
  3. Na seção "Header", altere o campo "Logo Image Src". Mude de "gif" para "png".
  4. Clique em "Save Config".

Para instalar o logotipo de sua empresa, você precisa enviar a imagem do logo para a pasta /images do seu tema do Magento.

 

3. Configurar páginas

Para fazer com que seu template fiquei idêntico à nossa demonstração, você precisa criar páginas de forma específica. Providenciamos todos os arquivos com o código fonte das páginas. Você precisa adicioná-las ao painel do Magento seguindo as instruções abaixo. A tabela de configurações está localizada na página Technical details na demonstração do template. O arquivo de documentação contém o link para esta página:

  • documentation
    • documentation.html

Exemplo de tabela de configurações:

# Page Name - Nome da página Page HTML (aba Content) Page XML (aba Design) Layout
1. Home Page - Página principal sources\demo\pages\home_page.phtml sources\demo\pages\home_page.xml 1 column - 1 coluna
2. About Us - Sobre nós sources\demo\pages\about_us.phtml NA 1 column - 1 coluna
3. Customer Service - Atendimento (SAC) sources\demo\pages\customer_service.phtml NA 1 column - 1 coluna
4. Privacy Policy - Política de Privacidade sources\demo\pages\privacy_policy.phtml NA 1 column - 1 coluna

A página principal do template é a que mais será afetada. Em geral ela contém os seguintes elementos: banners, slider, galeria, etc.

Nós recomendamos que você desabilite o editor WYSIWYG. Para fazer isso, navegue para System > Configuration. Abra a seção "Content Management" e na janela WYSIWYG Options troque a opção "Enable WYSIWYG Editor" para "Disabled Completely".
Se você já tiver feito mudanças nos códigos das suas páginas e preferir adicionar nosso código sem afetar suas mudanças, recomendamos entrar em contato com profissionais da área ou faça um backup antes de alterar.
  1. No menu do topo, selecione CMS > Pages. Aqui você pode encontrar todas as páginas da loja. Home page (a página inicial), About Us (sobre nós) e Customer Service (SAC) já devem estar habilitadas, então você não precisa adicioná-las. Você só precisa atualizar o HTML e XML delas. Abra qualquer uma delas para acessar o conteúdo.
  2. Abra a página e clique na aba Content. Ela contém o código HTML da página. Você pode pegar o código HTML do arquivo fonte apropriado. VOcê pode ver a lista de arquivos fonte incluídos na seção "Template Settings" da página de visualização do template. Copie o código do arquivo e cole na área content.
  3. Quando terminar de mexer no código HTML, mude para a aba "Design". Nela, você encontrará código XML. Esse código permite que você adicione blocos estátios do Magento e widgets na página. O código XML deve ser copiado da mesma maneira que o HTML. Copie o código na área "Layout Update XML".
  4. Não esqueça de escolher uma opção válida de "Layout".
  5. Se você precisar de uma página nova, cique no botão "Add New Page" no topo à direita.

 

Se tiver dificuldade, leia um tutorial detalhado em How to configure pages manually.

4. Configurar bloco estático

Quando terminar de configurar as páginas, você vai precisar configurar os "static blocks" (blocos estáticos). O procedimento é parecido: você precisa copiar o código fonte e colar nos campos correspondentes do painel administrativo do Magento. Você vai precisar de uma tabela parecida, mas para blocos estáticos.

  1. Do menu superior, escolha CMS > Static Blocks.
  2. Clique em "Add New block" no canto direito para criar um novo bloco.
  3. Escreva o nome e identificador do bloco. Você pode ver detalhes do bloco na tabela da página de demonstração do template (veja na parte de configurações da página).
  4. Abra o arquivo fonte do bloco e copie o HTML na área do conteúdo.

 

Perceba que: alguns dos templates não tem blocos estáticos, então essa tabela pode estar vazia.
Os arquivos fonte com os blocos estáticos estão em "sources\demo\static_blocks\".

Quando terminar de configurar as páginas e os blocos, sua loja está pronta para abrir.

Configurar extensões

1. Módulo de moeda

  1. Abra o admin do Magento.
  2. No menu, escolha System > Configuration.
  3. Então clique em General e depois na aba Currency Setup.
  4. Em Currency Options, segurando a tecla CTRL escolha as moedas permitidas que podem ser usadas para mostrar os preços na sua loja do Magento. Quando terminar, clique em Save Config para guardar suas alterações.
  5. Navegue até System > Manage Currency Rates no menu superior.
  6. Defina o valor da proporção das moedas ou utilize o Import Service. Clique em Save Currency Rates para alterá-las.

2. Módulo de pesquisa de opinião

Do menu superior, vá para CMS > Polls.

6. Política de Cookies

Cookies são pequenos arquivos de texto que sites colocam no computador dos visitantes. Geralmente servem para identificar um visitante e prover uma experiência melhor. Cookies são uma parte crucial de muitas funções em sites que costumamos tomar como triviais.

Por exemplo, em geral usamos cookies para que as lojas online lembrem o que adicionamos ao carrinho de compras. Também são usados para mantê-lo logado em um site ou manter estatísticas de uso e informações para os donos do site.

Se você tem um site, quase certeza que usa cookies. Provavelmente para monitorar o número e o comportamento dos seus visitantes através do Google Analytics, por exemplo. Também podem ser usados para mostrar anúncios relevantes para visitantes. Cookies também empoderam partes chave do seu sistema de compras online.

26 de Maio de 2012 - A ICO lançou uma mudança na política de Cookies. Iniciou-se daquele momento em diante permitiu-se um "consentimento tácito" em relação ao uso de cookies, se você mantiver às claras onde encontrar suas políticas de privacidade/cookies.

Seguindo as condições da Cookie Law, você deve manter módulos em seu site para mostrar mensagens aos seus usuários sobre o uso de Cookies no seu site. Nós recomendamos os seguintes:

  • Creare EU Cookie Law Banner
  • EU Cookie Law Compliance - Implied Consent
