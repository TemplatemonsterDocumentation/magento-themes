Introduzione Cosa trovi in questo pacchetto e a cosa serve

Grazie per aver acquistato un tema Magento. Questa documentazione si suddivide in diverse parti e illustra l'intera procedura per l'installazione e la gestione di un negozio Magento a partire da zero. Abbiamo fatto del nostro meglio per rendere questo manuale il più chiaro e facile da seguire possibile.

Cos'è Magento Commerce?

Magento è una piattaforma eCommerce ricca di funzioni sviluppata utilizzando tecnologie open-source. La piattaforma offre al commerciante online un livello senza precedenti di flessibilità e controllo su aspetto, contenuti e funzionalità del suo negozio eCommerce. L'interfaccia di amministrazione di Magento è dotata di potenti strumenti di marketing, SEO e gestione catalogo che danno al commerciante la possibilità di creare siti su misura per i suoi bisogni specifici. Sviluppato per essere completamente scalabile e per godere della rete di supporto Varien, Magento offre a ogni compagnia la soluzione eCommerce definitiva. Altre informazioni

Cos'è un tema Magento?

Un tema Magento è un'interfaccia personalizzato per il tuo negozio Magento. In altra parole, installando un nuovo tema puoi facilmente modificare l'aspetto del tuo negozio Magento. Pur con tutta la sua semplicità, un tema Magento è distribuito con tutti i file sorgente inclusi, che l'utente può modificare o espandere secondo le proprie esigenze.

Struttura File

Il pacchetto del tema che hai scaricato è struttutato in diverse cartelle. Vediamo cosa ciascuna di esse contiene:

  • screenshots - contiene delle schermate del tema. Da non usare per la produzione
  • sources - contiene i file sorgente del tema
    • psd - i file sorgente per Adobe Photoshop (.psd)
    • demo - contiene pagine aggiuntive e blocchi custom forniti insieme al tema
    • sample_data - contiene file campione del tema Magento
      • media - contiene file e directory da caricare sulla directory root della tua installazione Magento
      • dump.sql.gz - Il file di backup del database. Contiene contenuti campione del tema.
  • theme### - contiene file e directory del tema Magento da caricare nella directory root della tua installazione Magento
  • fonts_info.txt - contiene i link da cui scaricare i font custom del template
  • info.txt - contiene istruzioni su come estrarre i file sorgente.

Preparazione

Per immagazzinare dati, il motore e-commerce Magento ha bisogno di MySQL. Prima di procedere con l'installazione di Magento, assicurati di aver già creato il database MySQL. Lo puoi fare attraverso il pannello di controllo del tuo host, oppure con lo strumento phpMyAdmin.

Puoi anche vedere la guida dettagliata su come creare un database MySQL. Nel caso ci siano problemi, si prega di contattare il proprio provider, per ulteriore assistenza.

Durante la procedura di creazione del database MySQL, otterrai i dettagli del database, come: nome database, nome utente, password e server database. Annota questi dettagli, perché ti serviranno durante l'installazione di Magento.

Installazione motore Magento

1. Requisiti Server

Prima di tutto assicurati che il tuo server risponda ai requisiti di Magento:

  1. Sistema: Unix, Linux o Windows. Si consiglia caldamente di usare Unix.
  2. Web server: Apache Web server 1.3 o successivi, Microsoft IIS Web server 6.0 o successivi e nginx 1.0 o successivi.
  3. PHP 5.2* o successivi.
  4. MySQL 5.0 o successivi.
  5. Almeno 32 Mb di RAM sul tuo server (raccomandati 64 Mb; più sono meglio è).
Magento può funzionare con PHP 5.1 o successivi, ma le versioni precedenti alla 5.2 hanno dei bug che possono impedire il corretto funzionamento di determinate funzioni (ad esempio problemi di fuso orario non valido).

2. Scaricare

Per installare Magento devi scaricare il pacchetto motore Magento dal sito ufficiale all'indirizzo www.magentocommerce.com/download.

Inserisci il tuo nome e e-mail nella casella Download e clicca il pulsante 'Download'.

Assicurati che la tua versione del motore Magento risponda ai requisiti del template. Tutte le versioni di Magento sono disponibili su SourceForge.

3. Estrarre i File

Una volta completato il download, dovrai estrarre i file del pacchetto motore Magento.

Lo si può fare utilizzando qualunque strumento di gestione archivi, come WinZip(PC), StuffitExpander(MAC), ecc.

Questi tutorial spiegano in maggiore dettaglio come estrarre file con WinZip (PC) e StuffitExpander (MAC).

Nota bene: se il pannello di controllo del tuo host ha l'opzione di estrarre i file, puoi usare direttamente quello e estrarre i file direttamente sul tuo server dopo l'upload.

4. Caricare i file del motore Magento

Se hai estratto i file localmente sul tuo computer, dovrai ora caricarli sul server.

Lo si può fare utilizzando il File Manager del pannello di controllo oppure con un'applicazione esterna per gestione FTP, come FileZilla, TotalCommander, CureFTP, ecc.

Qui puoi trovare un tutorial video dettagliato su come caricare dei file su di un server utilizzando un gestore FTP e su come caricare dei file su di un server utilizzando cPanel (WebHost Manager, WHM).

5. Installazione motore Magento

Una volta finito l'upload dei file, puoi iniziare la procedura di installazione del motore Magento. Apri il tuo browser e digita il tuo 'nome di dominio o percorso alla tua directory Magento' nella barra degli indirizzi. Adesso dovrebbe apparire una schermata di benvenuto. Segui le istruzioni per installare Magento.

Nota bene: il file dump.sql.gz va importato nel database pulito PRIMA di installare Magento.

Il pacchetto del template Magento contiene i file di dati campione (dump.sql o dump.gz). Questo file si trova nella directory /sources/sample_data del tuo pacchetto template. Questo file installa i dati campione e fa in modo che il tuo sito appaia proprio come nell'anteprima dimostrativa del template.

Per installare il file dump, fai così:

  1. Apri il tuo strumento di gestione database (PhpMyAdmin). Vi si può accedere attraverso il pannello di controllo dell'host.
  2. Crea un nuovo database.
  3. Seleziona il database creato per il CMS Magento.
  4. Seleziona l'opzione "IMPORT" e clicca sul pulsante "Browse".
  5. Vai sulla cartella "/sources/sample_data" del template e seleziona il file dump.sql.gz.
  6. Premi il pulsante "GO".
Non usare il file dump.sql.gz se hai già un negozio in funzione. Importando il file dump.sql.gz nel tuo database attivo i tuoi prodotti verrebbero rimossi. Se non vuoi installare i contenuti campione del template, puoi installare Magento sul database 'pulito'. Salta la fase di installazione dei dati campioni.

Installazione template Magento

La fase successiva della procedura di setup è l'installazione del template. Il template va installato in ogni caso, che tu voglia o meno i dati campione.

  1. Prima di tutto, apri il pacchetto template e vai sulla cartella theme###.
  2. Carica i contenuti della cartella theme### nella directory di installazione Magento (carica "app," "skin" e altre cartelle). Queste cartelle contengono i file del tema. Questo non avrà nessun impatto sui file esistenti.
  3. Se hai importato il file dump.sql.gz e vuoi usare i prodotti campione del template, devi caricarne le immagini. Carica la cartella "media" dalla directory sources/sample_data nella cartella root di installazione Magento.
  4. Se non hai installato il file dump.sql.gz file, non c'è bisogno di caricare la cartella "media".

Fase 1: Assistente Installazione

Leggi il contratto di licenza e seleziona la casella "I agree to the above terms and conditions", se accetti i termini.

Fase 2: Assistente Installazione (Localizzazione )

Seleziona località, fuso orario e valùta predefiniti. L'opzione località (Locale) definisce la lingua di backend del tuo negozio Magento. Se vuoi cambiare la lingua frontend, devi scaricare e installare altri Locale frontend di Magento. Tutte le opzioni possono facilmente essere cambiate in seguito attraverso il pannello di controllo di Magento.

Fase 3: Configurazione

Ora devi inserire i dettagli del database del negozio e selezionare alcune impostazioni iniziali. Database Connection – inserisci il nome di host del database, nome utente, password e nome. Contatta il tuo provider se non hai tutti i dettagli necessari.

Assicurati di avere un database già creato per l'installazione di Magento. E assicurati anche che sia 'pulito', senza tabelle. Web access options e Session Storage Options – se non sei sicuro, lascia queste opzioni così come sono. Le impostazioni di default solitamente funzionano bene.

Se tutto è a posto, clicca il pulsante 'Continue'.

Fase 4: Create Admin

In questa fase devi inserire dettagli personali del proprietario e amministratore del sito.

Una volta fatto, clicca il pulsante 'Continue'.

Fase 5: You're All Set!

Questa è la fase finale dell'installazione. Copia la chiave di cifratura generata e conservala al sicuro. Per vedere il front end del negozio o il pannello di amministrazione, puoi usare i pulsanti nell'angolo in basso a destra.

Installare il Template Magento su di un sito web esistente

1. Configurazione manuale del template

Prima di tutto, assicurati che la tua versione di Magento risponda ai requisiti del template. Se stai usando una versione più vecchia di Magento, dovrai aggiornarla. Non dimenticare di fare una copia di back up dei file e del database. Le istruzioni per l'aggiornamento di Magento sono disponibili sul sito Magento Wiki.

Se il tuo motore Magento risponde ai requisiti del template, puoi iniziare con l'installazione.

  1. Apri il pacchetto del template e vai sulla cartella theme###.
  2. Carica i contenuti della cartella theme### sulla directory di installazione Magento (carica "app," "skin" e altre cartelle). Queste cartelle contengono i file del tema e non avranno nessun impatto sui tuoi file esistenti.

1. Configurazione Template

In questa fase si può configurare il template manualmente.

  1. Attiva nuovo tema
    • Apri il pannello di amministrazione di Magento e nel menù in alto seleziona System > Design.
    • Clicca sul pulsante "Add Design Change".
    • Nel campo "Custom Design", seleziona il nome del tema (in questo caso theme###).
    • Clicca sul pulsante "Save" per confermare le modifiche.
  2. Reindicizza dati
    • Apri il pannello di amministrazione di Magento e nel menù in alto seleziona System > Index Management.
    • Seleziona la casella "Select All" nell'angolo in alto sinistra.
    • Poi sulla destra, nel campo "Actions", seleziona "Reindex Data".
    • Clicca sul pulsante "Submit".
    • Sii paziente, potrebbe volerci un po' di tempo.

2. Configurazione Logo.

Segui questa procedura se ha problemi con l'immagine del logo.

  1. Apri il pannello di amministrazione di Magento e nel menù in alto seleziona System > Configuration.
  2. Seleziona la scheda "Design".
  3. Nella sezione "Header" modifica il valore del campo "Logo Image Src". Cambialo da "gif" a "png."
  4. Clicca sul pulsante"Save Config".

3. Configurazione Pagine

La home page del template può contenere oggetti di layout come banner, slider, galleria, animazioni in flash, ecc. Per aggiungere blocchi al tuo negozio, segui queste istruzioni.

E' consigliato disabilitare l'editor WYSIWYG. Per farlo, seleziona System > Configuration dal menù admin, in alto. Apri la sezione "Content Management" e nella finestra WYSIWYG Options imposta l'opzione "Enable WYSIWYG Editor su Disabled Completely."
  1. Dal menù in alto, seleziona CMS > Pages. Le pagine Home page, About Us e Customer Service dovrebbero già essere disponibili, quindi non dovrai aggiungerle. Devi solo aggiornare i contenuti HTML e XML delle pagine.
  2. Se devi aggiungere una nuova pagina, clicca sul pulsante "Add New Page" all'angolo in alto a destra.
  3. Apri la pagina e seleziona la scheda Content. La scheda Content contiene il codice HTML della pagina. Puoi prelevare il codice HTML dal file sorgente appropriato. Nella tabella che segue puoi vedere una lista dei file sorgente. Copia il codice dal file sorgente e poi incollalo nell'area contenuti.
  4. Fatto questo, passa a "Design." Design contiene il codice XML della pagina. Il codice va copiato nello stesso modo in cui si è copiato il codice HTML dal file sorgente. Copia il codice XML all'interno dell'area "Layout Update XML".
  5. Non dimenticare di selezionare l'opzione "Layout" valida.

In caso di difficoltà, puoi seguire questa guida esaustiva su come congigurare le pagine manualmente.

# Nome Pagina HTML Pagina (Scheda Content) XML Pagina (Scheda Design) Layout
1. Home Page sources\demo\pages\home_page.phtml sources\demo\pages\home_page.xml 1 colonna
2. About Us sources\demo\pages\about_us.phtml ND 1 colonna
3. Customer Service sources\demo\pages\customer_service.phtml ND 1 colonna
4. Privacy Policy sources\demo\pages\privacy_policy.phtml ND 1 colonna

4. Configurazione Blocchi Statici

Una volta terminata la configurazione delle pagine, dovrai configurare i blocchi statici. La procedura è simile: devi semplicemente copiare il codice dal file sorgente.

  1. Dal menù in alto, seleziona CMS > Static Blocks.
  2. Clicca sul pulsante "Add New block" nell'angolo a destra per aggiungere un nuovo blocco.
  3. Inserisci il nome e l'identificativo del blocco. Puoi trovare i dettagli del blocco nella tabella qui sotto.
  4. Apri il file sorgente del blocco e copia il codice HTML nell'area contenuti.

Nota Bene: alcuni template non hanno blocchi statici, quindi la tabella potrebbe essere vuota.
# Nome Blocco Titolo Blocco Identificatore Contenuti
1. Footer List Footer List footer_list sources\demo\static_blocks\footer_list.phtml
2. Custom tab Custom tab custom_tab sources\demo\static_blocks\custom_tab.phtml
3. contact_block contact_block contact_block sources\demo\static_blocks\contact_block.phtml
4. Map contents Map content map_content sources\demo\static_blocks\map_content.phtml
5. Footer info Footer info footer_info sources\demo\static_blocks\footer_info.phtml
I file sorgente con il codice dei blocchi statici sono contenuti nella directory "sources\demo\static_blocks\" del pacchetto.

Una volta terminata la configurazione di pagine e blocchi, il tuo negozio è pronto a partire.

2. Personalizzazione Generale

Quando si modificano le pagine e i blocchi statici, è consigliato disabilitare l'editor WYSIWYG. Per farlo, seleziona System > Configuration dal menù admin, in alto. Apri la sezione "Content Management" e nella finestra WYSIWYG Options imposta l'opzione "Enable WYSIWYG Editor su Disabled Completely."

1. Come modificare le pagine

  1. Dal menù in alto, seleziona CMS > Pages. Qui sono disponibili tutte le pagine del negozio. Aprine una per accedere ai suoi contenuti.

  2. Dalla colonna di sinistra, seleziona la scheda Content. Questa contiene il codice HTML della pagina. Sei libero di modificare il codice secondo le tue esigenze.
  3. Passa alla scheda "Design" per accedere alla pagina contenuti XML. Il codice XML ti permette di incorporare blocchi statici o Widget di Magento all'interno della pagina.

2. Come modificare i blocchi statici

Dal menù in alto, seleziona CMS > Static Blocks.

Per fare un esempio, proviamo ad aprire un blocco statico. Ora potrai vederne il contenuti in HTML. In questo esempio stiamo usando il blocco banner. Usando il codice HTML puoi facilmente modificare l'immagine e il link del banner.

3. Come configurare il modulo valùta

  1. Apri il pannello di amministrazione Magento.
  2. Dal menù in alto, seleziona System > Configuration.
  3. Poi clicca su General e seleziona la scheda Currency Setup.
  4. In Currency Options, tenendo premuto il pulsante CTRL, seleziona le valute da utilizzare per visualizzare i prezzi nel tuo negozio Magento. Fatto ciò, clicca sul pulsante Save Config per salvare le modifiche.
  5. Poi vai su System > Manage Currency Rates dal menù in alto.
  6. Imposta i valori di scambio della valuta, oppure utilizza il servizio di importazione (Import Service). Clicca sul pulsante Save Currency Rates per salvare le modifiche.

4. Come modificare il modulo sondaggi (Polls)

Dal menù in alto, seleziona CMS > Polls.

Configurazione Estensioni

1. Olark Live Chat

Per impostare una chat dal vivo come quella della Demo, dovrai iscriverti al sito web ufficiale di Olark. Dopo aver fornito tutti i dati necessari, otterrai il codice da aggiungere attraverso il pannello di amministrazione.

2. Cloud Zoom

Le impostazioni Cloud Image Zoom ti permettono di modificare le dimensioni dell'immagine di pop-up e di impostare le dimensioni originali dell'immagine.

3. Aggiungi

Le impostazioni di Sharing Tool ti permettono di determinare in che modo le icone debbano essere visualizzate. Le impostazioni consigliate sono quelle che vedete nell'immagine.

4. Informativa sui Cookie

I cookie sono dei piccoli file di testo che un sito web crea sul computer di un visitatore. Vengono utilizzati tipicamente per identificare quello specifico visitatore allo scopo di fornirgli un'esperienza di navigazione migliore. I Cookie sono un elemento importantissimo per il corretto funzionamento di tante funzioni dei siti web che tutti ormai diamo per scontate.

Ad esempio, sono solitamente proprio i cookie che permettono ai negozi online di ricordare quali articoli hai messo nel tuo carrello. Sono usati anche per tenere attivo il tuo login sul sito, o per dare ai gestori del sito informazioni statistiche importanti.

Se gestisci un sito web, esso quasi sicuramente usa i cookie. La loro funzione più probabile è quella di monitorare il numero dei visitatori e i loro comportamenti, servendosi di strumenti come Google Analytics. Possono essere usati anche per mostrare agli utenti delle pubblicità il più possibile rilevanti, oppure - per chi vende prodotti - per far funzionare alcuni tra gli elementi più importanti del sistema di compravendita del tuo negozio online.

26 maggio 2012 - La ICO ha pubblicato una modifica all'applicazione della regolamentazione: "il consenso implicito" è ora consentito purché sia ben chiaro dove si trovano le tue informative su cookie e privacy.

Per aderire alle direttive della legge sui Cookie, dovrai aggiungere moduli appropriati sul tuo sito. Il modulo mostrerà un messaggio di avviso riguardo l'utilizzo dei cookie sul tuo sito web. Puoi usare questi::

  • Creare EU Cookie Law Banner
  • EU Cookie Law Compliance - Implied Consent

Conclusioni

Per gli usi più avanzati, serve molto di più di quanto esposto in questo manuale.
